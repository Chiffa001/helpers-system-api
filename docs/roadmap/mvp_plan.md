# MVP Plan

## Этап 0 — Ядро и workspace
- auth (Telegram initData)
- модель User без глобальной роли
- Workspace и WorkspaceMember
- роли: super_admin, workspace_admin, assistant, client
- создание и управление workspace-ами (super_admin)
- выбор активного workspace при входе

## Этап 1 — Проекты и участники
- проекты внутри workspace
- добавление клиентов и помощников в проект
- специализации помощников

## Этап 2 — Онбординг клиента
- заявки клиентов (client applications) с привязкой к workspace
- flow подключения через бота
- первый счет и подтверждение оплаты workspace_admin-ом

## Этап 3 — Мероприятия
- создание мероприятий
- подтверждение участия (бесплатное и платное)
- выставление счетов на мероприятие

## Этап 4 — Счета и оплаты (fiat)
- выставление произвольных счетов
- оплата по ссылке
- ручное подтверждение оплаты workspace_admin-ом

## Этап 5 — Программа сопровождения
- создание программы проекта
- добавление шагов с привязкой к клиенту, направлению, помощнику
- счета на шаги

## Этап 6 — Прогресс
- прогресс по проекту
- прогресс по клиентам и направлениям
- % выполнения

## Этап 7 — Crypto
- TON и USDT (TON) через CryptoBot
- crypto invoice
- webhook подтверждение

---

## MVP готов если:
- можно создать workspace и назначить admin-а
- клиент может подключиться через бота к нужному workspace
- есть проекты с клиентами и помощниками
- есть мероприятия с оплатой
- есть программа сопровождения
- есть прогресс
- работают счета (fiat и crypto)
- данные между workspace-ами полностью изолированы
