# Screens: Workspace

Экраны, связанные с выбором, созданием и управлением workspace-ами.
Покрывают Этап 0 MVP: auth, роли, выбор и создание workspace.

---

## 1. Экран выбора workspace

**Когда показывается:** пользователь авторизован, но принадлежит к нескольким workspace-ам.
**Когда пропускается:** пользователь состоит только в одном workspace — сразу переход на главную.

### Layout
- Шапка: логотип или название приложения, без кнопки назад
- Подзаголовок: "Выберите рабочее пространство"
- Список карточек workspace-ов (вертикальный скролл)
- Футер: ссылка "Нет нужного пространства? Свяжитесь с администратором"

### Карточка workspace
```
┌─────────────────────────────────────┐
│  [иконка/аватар]  Название          │
│                   Роль: Admin        │
│                   3 проекта · 12 кл. │  ← только если есть данные
└─────────────────────────────────────┘
```
- иконка: первая буква названия или загруженный логотип
- название workspace
- роль пользователя в этом workspace (badge: Admin / Assistant / Client)
- краткая статистика (только для admin и assistant)
- вся карточка — кликабельна

### Состояния
- **загрузка:** скелетон-список из 2–3 карточек
- **пустое:** не должно возникать (если пользователь ни в одном workspace — показывается заглушка "Вы не подключены ни к одному пространству. Обратитесь к администратору.")

## 2. Экран создания workspace (super_admin)

**Кто видит:** только super_admin.
**Откуда доступен:** системный раздел → кнопка "Создать пространство".

### Layout
- Шапка: "Новое рабочее пространство" + кнопка назад
- Форма
- Кнопка "Создать" (основное действие, внизу экрана)

### Поля формы

| Поле | Тип | Обязательное | Комментарий |
|------|-----|--------------|-------------|
| Название | text input | да | например, "Агентство Иванова" |
| Slug | text input | да | уникальный ID, только латиница и дефис; автозаполняется из названия |
| Администратор | user picker | нет | поиск по Telegram username или ID; можно добавить позже |

### Поле Slug
- автозаполняется транслитерацией из названия ("Агентство Иванова" → "agentstvo-ivanova")
- доступно для ручного редактирования
- inline-валидация: проверка уникальности при потере фокуса
- под полем: подсказка "Используется для идентификации пространства. Изменить после создания нельзя."

### Состояния кнопки "Создать"
- неактивна, пока не заполнены обязательные поля
- loading-состояние во время запроса
- при ошибке (slug занят) — inline-ошибка под полем slug, кнопка снова активна

### После создания
- переход на экран управления созданным workspace
- toast: "Пространство создано"

## 3. Экран управления workspace (super_admin)

**Откуда:** список всех workspace-ов → карточка → "Управление".

### Layout
- Шапка: название workspace + badge статуса (Active / Suspended) + кнопка назад
- Секции: Основное, Участники, Биллинг, Опасная зона

### Секция: Основное
- Название (редактируемое)
- Slug (только отображение, не редактируется)
- Статус: Active / Suspended / Archived — выпадающий список

### Секция: Участники
- список workspace_admin-ов (имя, username Telegram, badge "Admin")
- кнопка "Добавить администратора" → модальное окно с поиском пользователя

### Секция: Биллинг
- текущий тариф (badge: Free / Basic / Pro / Business)
- дата следующего списания (если есть активная подписка)
- общая сумма удержанных комиссий (если Free)
- кнопка "Изменить тариф"

### Секция: Опасная зона
- кнопка "Приостановить" (если Active) — требует подтверждения
- кнопка "Восстановить" (если Suspended)
- кнопка "Архивировать" — требует подтверждения + ввода slug

### Модальное окно подтверждения (Приостановить / Архивировать)
```
Приостановить пространство?

Доступ для всех участников будет
заблокирован до восстановления.

[ Отмена ]   [ Приостановить ]
```
Для архивирования — дополнительно поле "Введите slug для подтверждения".

## 4. Список всех workspace-ов (super_admin)

**Откуда:** системный раздел / главная super_admin.

### Layout
- Шапка: "Рабочие пространства" + кнопка "Создать"
- Фильтры: All / Active / Suspended / Archived (таб-переключатель)
- Список карточек
- Счётчик: "12 пространств"

### Карточка в списке
```
┌─────────────────────────────────────────┐
│ Название workspace         [Active]     │
│ slug: ivanov-agency                     │
│ 3 admin · 8 assistant · 24 client       │
│ Тариф: Pro · Следующее списание: 1 апр  │
└─────────────────────────────────────────┘
```
- нажатие → переход на экран управления workspace

### Пустое состояние
- иконка + текст "Нет рабочих пространств"
- кнопка "Создать первое"

## 5. Настройки workspace (workspace_admin)

**Откуда:** раздел настроек текущего workspace.

### Layout
- Шапка: "Настройки пространства" + кнопка назад
- Секции: Основное, Участники, Биллинг

### Секция: Основное
- Название (редактируемое)
- Slug (только отображение)
- Кнопка "Сохранить" (появляется при изменениях)

### Секция: Участники
- список всех участников с ролью
- поиск по имени
- кнопка "Пригласить" → модальное окно
- для каждого участника: кнопка "..." → изменить роль / удалить

### Модальное окно: Пригласить участника
- поиск по Telegram username
- выбор роли (workspace_admin / assistant)
- кнопка "Пригласить"
- клиенты добавляются только через заявки, не через это окно

### Секция: Биллинг
- текущий тариф + описание лимитов
- использование: X из Y клиентов, X из Y проектов (прогресс-бар)
- сумма удержанных комиссий за текущий месяц (если Free)
- кнопка "Улучшить тариф" (если Free или Basic)

## 6. Экран выбора тарифа

**Откуда:** Настройки → Биллинг → "Улучшить тариф" или уведомление о лимите.

### Layout
- Шапка: "Тарифный план" + кнопка назад
- Карточки тарифов (вертикально)
- Текущий тариф помечен "Текущий план"

### Карточка тарифа
```
┌──────────────────────────────────────┐
│  Pro                    [Текущий]    │
│  X ₽ / месяц                         │
│                                      │
│  ✓ до 100 клиентов                   │
│  ✓ неограниченные проекты            │
│  ✓ crypto оплаты                     │
│  ✓ без комиссии с транзакций         │
│                                      │
│  [ Выбрать ]                         │
└──────────────────────────────────────┘
```
- Free: кнопка недоступна (уже используется по умолчанию)
- текущий: кнопка заменена на "Текущий план"

### Примечание под списком
"На тарифе Free с каждой транзакции удерживается X% в пользу платформы."

---

## Общие правила для этих экранов

- все экраны workspace-уровня доступны только после выбора активного workspace
- super_admin видит системные экраны (пп. 3–4) поверх workspace-экранов
- переключение workspace — всегда доступно через иконку/название в шапке или через отдельный пункт меню
- смена workspace не требует повторной авторизации
