# Roles

## Уровни ролей

Роли разделены на два уровня: системный и уровень workspace.

```
super_admin                     ← системный уровень
    └── workspace_admin         ← уровень workspace
            ├── assistant       ← уровень workspace
            └── client          ← уровень workspace
```

Один пользователь Telegram может иметь разные роли в разных workspace-ах.
Например: workspace_admin в одном бизнесе и client в другом.

---

## Super Admin

Системная роль. Назначается вручную при инициализации системы.

Может:
- создавать новые workspace-ы
- назначать workspace_admin-ов
- приостанавливать и архивировать workspace-ы
- видеть список всех workspace-ов и их статусы
- заходить в любой workspace в режиме просмотра

Не может:
- управлять проектами и клиентами напрямую (это зона workspace_admin)

---

## Workspace Admin

Полный контроль внутри своего workspace. Аналог прежней роли admin, но ограничен своей площадкой.

Может:
- управлять пользователями своего workspace
- создавать и архивировать проекты
- добавлять новых клиентов через заявку на подключение
- подтверждать подключение клиента после первой оплаты
- назначать клиентов в проекты
- назначать помощников в проекты
- задавать роли помощников внутри проекта
- управлять оплатами
- видеть все данные своего workspace
- приглашать новых assistant-ов в workspace

Не может:
- управлять другими workspace-ами
- видеть данные других workspace-ов

---

## Assistant

Может:
- создавать мероприятия
- работать в проектах своего workspace, куда он назначен
- вести общую программу сопровождения проекта
- вести свои направления работы внутри проекта
- работать с одним или несколькими клиентами в рамках проекта
- выставлять счета

Не может:
- управлять проектами, куда он не назначен
- менять финальные статусы оплат
- управлять участниками workspace

---

## Client

Может:
- оставить заявку на подключение через Telegram
- видеть свои проекты (в рамках одного workspace)
- видеть общие данные проекта, доступные клиенту
- видеть свои мероприятия
- оплачивать
- видеть программу проекта
- видеть свой персональный прогресс и общий прогресс проекта

---

## Дополнительные уточнения
- один проект может содержать нескольких клиентов
- один проект может содержать нескольких помощников
- один помощник может отвечать только за своё направление, например медицину или документы
- один клиент может состоять в одном активном проекте на старте MVP (упрощение)
- пользователь может быть членом нескольких workspace-ов с разными ролями
