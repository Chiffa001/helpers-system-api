# User Flows

## 0. Выбор workspace

При открытии Mini App:
1. backend определяет, к каким workspace-ам принадлежит пользователь
2. если workspace один — открывается сразу
3. если workspace несколько — показывается экран выбора площадки
4. выбранный workspace сохраняется как активный контекст сессии

---

## 1. Создание workspace (super_admin)

Super admin:
1. открывает системный раздел
2. создает новый workspace (название, slug)
3. назначает workspace_admin-а из существующих пользователей или по Telegram ID
4. workspace становится активным

---

## 2. Навигация по экранам

Пользователь:
1. открывает Mini App
2. при необходимости выбирает workspace
3. видит nav bar для переключения между основными экранами
4. переходит между разделами без повторного входа

Nav bar содержит:
- главная
- проекты
- клиенты
- мероприятия
- программа
- счета

Для workspace_admin дополнительно:
- пользователи
- заявки

---

## 3. Создание проекта

Workspace admin:
1. создает общий проект внутри текущего workspace
2. добавляет список клиентов
3. добавляет список помощников
4. при необходимости указывает специализацию помощников, например медицина или документы

---

## 4. Список клиентов

Assistant или Workspace admin:
1. открывает экран клиентов
2. видит список клиентов по доступным проектам текущего workspace
3. использует поиск по имени, username Telegram или проекту
4. использует фильтры по статусу, проекту, ответственному помощнику и направлению
5. открывает карточку клиента из результатов

---

## 5. Подключение нового клиента

Канал входа: клиент пишет в Telegram-бота, привязанного к конкретному workspace.

Client:
1. пишет в Telegram-бота
2. бот открывает Mini App с формой заявки в контексте нужного workspace
3. клиент заполняет базовые данные (имя, телефон)
4. получает ссылку на первый счет
5. оплачивает первый счет

Workspace admin:
1. видит новую заявку в статусе pending
2. при успешной оплате заявка автоматически переходит в статус paid
3. admin проверяет и подтверждает подключение (approve)
4. после подтверждения добавляет клиента в проект или создает новый проект

---

## 6. Мероприятие

Assistant:
1. создает мероприятие внутри проекта
2. указывает дату, время, описание, цену или бесплатное
3. выбирает, для всего проекта или только для части участников
4. отправляет ссылку

Client:
1. открывает Mini App
2. видит только доступные ему мероприятия
3. оплачивает или подтверждает участие

---

## 7. Бесплатное участие

1. пользователь нажимает Подтвердить
2. получает статус confirmed_free

---

## 8. Платное участие

1. создается invoice
2. пользователь оплачивает
3. статус -> paid
4. пользователь добавляется в участники

---

## 9. Сопровождение

Assistant:
1. создает программу проекта
2. добавляет шаги
3. указывает, шаг общий или относится к конкретному клиенту
4. при необходимости закрепляет шаг за конкретным помощником
5. отмечает шаг как выполненный (execution_status = done)

Client:
1. видит только релевантные ему шаги
2. оплачивает этапы

---

## 10. Прогресс

Система показывает:
- прогресс проекта целиком
- прогресс по каждому клиенту
- прогресс по каждому направлению
- оплачено
- осталось
- % выполнения

---

## 11. Истечение счета

1. счет создается со статусом issued
2. если клиент не оплатил в срок, статус меняется на expired (фоновая задача)
3. assistant или admin видит просроченный счет
4. admin может отменить или пересоздать счет вручную

---

## 12. Пример: несколько бизнесов

1. super_admin создает два workspace: Агентство А и Агентство Б
2. назначает разных workspace_admin-ов
3. каждый workspace_admin видит только свои проекты и клиентов
4. один и тот же помощник может быть членом обоих workspace с разными ролями
5. оплаты и прогресс полностью изолированы между workspace-ами

---

## 13. Пример: семейный проект

1. workspace_admin создает проект для мужа и жены
2. добавляет двух клиентов в один проект
3. добавляет помощника по медицинскому направлению
4. добавляет помощника по документам
5. медицинские шаги видит профильный помощник и нужный клиент
6. документные шаги видит профильный помощник и нужный клиент
7. общие оплаты и прогресс собираются на уровне проекта
